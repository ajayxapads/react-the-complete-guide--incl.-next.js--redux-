🚀 React Rendering Flow (Easy Notes)

📌 1. What you see in browser source code

When you inspect a React website source:

  • ❌ You do NOT see UI (header, text, images)
  • ✅ You only see HTML + JavaScript file

React builds UI using JavaScript dynamically.

The actual UI is created after JavaScript runs. :contentReference[oaicite:0]{index=0}

📌 2. Role of index.html

This is the main HTML file sent to users.

<div id="root"></div>

React injects the entire app inside this div.

📌 3. Entry Point (index.jsx)

This is the first file that runs in React.

  • Imports App component
  • Starts the React app

It is called the "entry point" of the app.

📌 4. Import & Export

import App from "./App";

👉 This is normal JavaScript, not React-specific.

📌 5. How React renders UI

const root = ReactDOM.createRoot(document.getElementById("root")); root.render(<App />);
  • Select root div
  • Create React root
  • Render App component

📌 6. What createRoot does

It connects React with the real DOM.

  • Takes existing HTML element
  • Makes it React root container

📌 7. What render() does

It displays the React component on screen.

  • Renders App component
  • Also renders all nested components

📌 8. Component Tree

React creates a hierarchy of components.

  • App (parent)
  • Header (child)
  • Other nested components

This structure is called Component Tree.

📌 9. Important Concept (Very Asked)

Custom components DO NOT appear in DOM.

  • Only real HTML elements appear
  • React converts components → HTML

📌 10. Uppercase vs Lowercase

  • Lowercase → HTML elements (div, header)
  • Uppercase → Custom components (App, Header)

👉 This helps React differentiate between them.

📌 11. How React Works Internally

  • Custom components → functions
  • React executes them
  • Gets JSX output
  • Converts to real HTML

React builds final DOM from all components combined.

📌 12. Final Understanding

  • React does NOT directly show components
  • It converts them into HTML
  • Then injects into root div

Components = Developer convenience DOM = Final output
