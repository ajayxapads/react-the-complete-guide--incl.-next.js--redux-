🚀 React Complete Guide (Easy + Detailed Notes)

📌 1. What is React?

React is a JavaScript library used to build user interfaces (UI), especially for websites.

Instead of writing everything in one file, React lets you break your UI into small reusable parts called components.

  • Fast because of Virtual DOM
  • Reusable components
  • Easy to manage UI

Example: Navbar, Footer, Card → all can be separate components.

📌 2. Components

Components are like functions that return HTML (JSX).

function App() { return <h1>Hello World</h1>; }

👉 You can reuse components anywhere in your app.

📌 3. JSX

JSX looks like HTML but it is actually JavaScript. (Used to describe & create HTML elements in JavaScript in a declarative way -- But browser cannot understand JSX directly, React comes with a build process that converts JSX to the code that actually runs in the browser.)

  • Use className instead of class
  • Wrap multiple elements in one parent
const element = <h1>Hello</h1>;

Component Functions Must Follow Two Rules

📌 Name Starts With Uppercase Character

  • The function name must start with an uppercase character
  • Multi-word names should be written in PascalCase (e.g., "MyHeader")
  • It’s recommended to pick a name that describes the UI building block (e.g., "Header" or "MyHeader")

📌 Returns “Renderable” Content

  • The function must return a value that can be rendered (displayed on screen) by React
  • In most cases, return JSX
  • Also allowed: string, number, boolean, null, array of allowed values

📌 4. Props

Props are used to pass data from parent component to child component.

function User(props) { return <h1>{props.name}</h1>; }

Props are read-only (you cannot change them).

📌 5. State

State is used to store data that changes over time (dynamic data).

import { useState } from "react"; const [count, setCount] = useState(0);
  • Updating state re-renders UI
  • Never change state directly

📌 6. Events

React handles events like clicks, input changes, etc.

<button onClick={handleClick}>Click</button>

👉 Always pass function reference, not function call.

📌 7. Conditional Rendering

Show different UI based on conditions.

{isLoggedIn ? <Dashboard /> : <Login />}

📌 8. Lists & Keys

Used to display multiple items dynamically.

{items.map(item => ( <li key={item.id}>{item.name}</li> ))}

Keys must be unique.

📌 9. Forms

React uses controlled components to manage form data.

<input value={name} onChange={(e) => setName(e.target.value)} />

📌 10. useEffect

useEffect is used to handle side effects like API calls, timers, etc.

useEffect(() => { console.log("Component Loaded"); }, []);
  • Runs after component renders
  • Empty array = run only once

📌 11. Custom Hooks

Custom hooks let you reuse logic across components.

function useFetch() { // reusable logic }

📌 12. Context API

Used to share data globally without passing props again and again.

const MyContext = React.createContext();

📌 13. Redux

Redux is used for global state management in large applications.

  • Store → holds data
  • Action → tells what to do
  • Reducer → updates state
const reducer = (state, action) => { if (action.type === "ADD") { return { ...state, count: state.count + 1 }; } };

📌 14. React Router

Used to create multiple pages without reloading.

<Route path="/about" element={<About />} />

📌 15. Performance Optimization

  • React.memo → prevents unnecessary re-render
  • useCallback → memoize functions
  • useMemo → memoize values

📌 16. API Calls

Used to fetch data from server.

fetch("https://api.com/data")
